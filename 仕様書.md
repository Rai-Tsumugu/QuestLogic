# SmartStudy Gate 仕様書
**Version 1.0.0**

## 1. 製品概要
### 1.1 製品名
SmartStudy Gate (スマートスタディ・ゲート)

### 1.2 コンセプト・キャッチコピー
「親の代わりにAIが宿題の『質』を評価し、物理デバイスが『約束』を守る学習習慣化システム」
『Smart Reward』『SmartStudy Quest』

### 1.3 開発背景・解決課題
- **親子間の対立解消**: 「宿題やったの？」という毎日の問い詰めによるストレスを軽減。
- **管理負担の低減**: 忙しい親に代わり、AIが学習の進捗と質を監視・評価。
- **学習の質の向上**: ただ「終わらせる」だけの形式的な実施（答えの丸写し等）を防ぎ、プロセスを評価。

## 2. ユーザー定義
### 2.1 ターゲットユーザー
- **親 (管理者)**:
    - 共働き等で忙しく、子供の勉強を常時見ることが難しい。
    - 子供のゲーム時間・スマホ利用時間を適切に管理したい。
    - 子供との無駄な衝突を避け、褒めて伸ばしたいと考えている。
- **子供 (利用者)**:
    - ゲームや動画視聴が好き。
    - 勉強を「やらされている」と感じており、モチベーションが低い。
    - 自分の頑張りを正当に評価してほしいと感じている。

## 3. 主要機能 (Core Features)

### 3.1 AI学習評価エンジン (Smart Analysis)
Gemini API等の画像認識技術を活用し、学習の「質」をスコアリングする。

- **Delta解析 (差分検知)**
    - **機能**: 宿題着手前（Before）と完了後（After）の写真2枚をアップロードし、その差分から実質的な作業量を算出。
    - **目的**: 「答えを写しただけ」「作業時間が短すぎる」などの不整合を検知。
- **プロセス評価**
    - **機能**: 以下の要素を検知し、加点対象とする。
        - 消しゴムの修正跡（試行錯誤の証拠）
        - 途中式の記述
        - 独自のメモや図解
        - 赤ペンによる解き直し
- **パーソナライズ学習 (AI Tuning)**
    - **機能**: 親がAIの採点結果を修正（「甘すぎる」「厳しくして」）すると、そのフィードバックを学習。
    - **効果**: 家庭ごとの教育方針（例：「字の綺麗さ」を最重視する、思考プロセスを重視するなど）に最適化された評価基準を構築。

### 3.2 報酬・制限システム (Reward & Restriction)
デジタルと物理の両面からアプローチする「アメとムチ」システム。

- **スマホアプリ制御 (Software Lock)**
    - **機能**: AIスコア（獲得ポイント）に基づき、iOS/AndroidのスクリーンタイムAPIを用いて特定アプリ（ゲーム、動画等）の利用制限を自動解除。
    - **ロジック**: 「宿題完了＝基本時間」＋「プロセス加点＝ボーナス時間」。
- **物理デバイス制御 (Hardware Lock)**
    - **機能**: NFC（おサイフケータイ機能等）を搭載した物理デバイスとの連携。
    - **仕組み**: アプリで報酬を獲得後、スマホをデバイスにかざすことで解錠。
- **段階的ペナルティプロトコル**
    1. **通知**: 時間外利用検知時、子供へ優しくリマインド。
    2. **警告**: 一定時間経過後、「親へ連絡する」旨を警告。
    3. **アラーム発動**: 大音量アラーム吹鳴。ICタグにかざすまで停止不可。
    4. **強制ロック**: 親の承認があるまでデバイスとスマホを完全ロック。

### 3.3 アプリケーション構成

#### 親用アプリ (Manager View)
- **ダッシュボード**: 今日の学習状況、AIスコア、子供の現在地（利用状況）を一目で確認。
- **学習レポート**: Before/After写真の比較閲覧、AIからの評価コメント、褒めポイントの提案。
- **チューニング設定**: 「字の丁寧さ」「思考プロセス重視」などのパラメータ調整、NG行動（丸写し等）の設定。
- **デバイス管理**: 強制ロック/解錠、緊急時の時間延長許可。

#### 子供用アプリ (SmartStudy Quest)
- **メイン画面**: RPG風UI。現在の「エネルギー（残り時間）」と「クエスト（宿題）」を表示。
- **撮影ミッション**: Before/After撮影をクエスト形式で実行。AIスキャン演出でワクワク感を創出。
- **リザルト画面**: 獲得ポイントの詳細（丁寧さボーナス、解き直しボーナス等）、AIコーチからのポジティブフィードバック。
- **解錠アクション**: スマホを物理デバイスにかざすアクションを「儀式化」。

## 4. ハードウェア仕様 (IoT Devices)
低コストかつ導入障壁の低いIoTデバイスを展開。

### 4.1 デバイスラインナップ
1. **Smart-Gate Box (収納ボックス)**
    - **概要**: ゲーム機（Switch等）やスマホを入れる透明/半透明ボックス。
    - **機能**: 蓋部分にNFCロック機構。中身が見えることで「あるのに使えない」心理的効果。
2. **Gate-Link (ワイヤーロック)**
    - **概要**: チャック付きケースや棚の取っ手をロックするワイヤー型。
    - **機能**: 既存の収納用品を活用可能。安価。
3. **Gate-Plug (電源制御アダプター)**
    - **概要**: 据え置き機（PS5等）やモニターの電源プラグに挟む。
    - **機能**: 通電のON/OFFを物理的に制御。

### 4.2 共通機能・仕様
- **通信**: NFC (Near Field Communication)
- **センサー**: 加速度・振動センサー（無理な破壊や持ち出しを検知し通知）
- **操作**: アプリ認証後のスマホタッチで解錠/施錠。施錠確認ログの送信。

## 5. ビジネスモデル
- **サブスクリプション (SaaS)**
    - 月額課金制。
    - 高度なAI解析、学習レポート、親へのコーチング機能を提供。
- **ハードウェア販売**
    - 導入ハードルを下げるため、原価に近い低価格で提供。
    - サービスの「フック」として活用。

## 6. MVP開発ロードマップ

### Phase 1: AI評価エンジンの確立 (1-2ヶ月)
- Gemini APIを用いた画像差分解析の実装。
- 学習プロセスの評価基準（プロンプト）の検証・作成。
- プロトタイプアプリ（写真撮影→判定→親確認）の開発。

### Phase 2: スマホ制限・報酬ロジックの実装 (3ヶ月目)
- iOS (Screen Time API) / Android (Digital Wellbeing) との連携検証。
- ポイント換算ロジックの実装。
- 子供向けフィードバックUIの実装。

### Phase 3: 物理デバイス連携 (4ヶ月目)
- 市販スマートロック等のAPI連携による検証。
- 自社デバイス（プロトタイプ）のNFC解錠テスト。

### Phase 4: βテスト・改善 (5-6ヶ月目)
- クローズドβテスト実施。
- チート手法（抜け道）の洗い出しと対策。
- 親によるAIチューニング機能の実装。

### 技術的課題 (Technical Risks)
- **OS制約**: iOS/Androidにおける他アプリ制御の権限範囲と審査基準。
- **精度**: 手書き文字認識や「丁寧さ」のような主観的評価の精度担保。
