親ダッシュボードでの表示、Before/After比較、承認アクションのテスト。

## テストケース

### 1. 表示確認 (Dashboard View)
- [ ] **TC-08-001**: 子供ステータス表示
    - **手順**: ダッシュボードを開く。
    - **期待値**: 子供の現在の状態（勉強中/完了/遊び中）が正しく表示されていること。
- [ ] **TC-08-002**: 通知からの遷移
    - **手順**: クエスト完了通知をタップ。
    - **期待値**: 解当するクエストの詳細承認画面が開くこと。

### 2. 承認アクション (Approval Action)

    - **手順**: 「承認」ボタンを押下。
    - **期待値**: ステータスが `approved` に更新され、画面上の表示も変わること（「承認済み」等）。
- [ ] **TC-08-005**: スタンプ送信
    - **手順**: スタンプを選択して送信。
