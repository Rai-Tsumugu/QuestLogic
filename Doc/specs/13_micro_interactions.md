# 13_micro_interactions.md

## 概要
ユーザーの操作に対するフィードバック（アニメーション、動き）の定義。
アプリの手触りを良くし、「生きている」感覚を与える。

## 1. ボタン・要素の動き
*   **Button Press**:
    *   `scale(0.95)`: 押した瞬間に少し縮む。
    *   RPG風ボタンの場合、擬似3Dの「押し込み」グラフィックを切り替える。
*   **List Item Tap**:
    *   Ripple Effect (波紋) または 背景色のハイライト。

## 2. ローディング演出
*   **Analysis Loading**:
    *   単なるスピナーではなく、「フクロウが本をめくっている」アニメーション。
    *   Text: 「解析中…」「呪文を解読中…」とメッセージを数秒ごとに切り替える。
*   **Image Upload**:
    *   プログレスバーが「剣のゲージ」になっている。

## 3. 画面遷移 (Transitions)
*   **Nav Push**:
    *   右からスライドイン（OS標準）ではなく、
    *   **Child**: 中央からのワイプ（昔のテレビ風）や、黒フェードアウト/イン。
    *   **Parent**: スムーズなフェード＆スライド。
*   **Modal Open**:
    *   下からポップアップし、背景をブラー（ぼかし）処理。

## 4. 数字・グラフのアニメーション
*   **Count Up**:
    *   スコア表示時、`0` から `85` までパラパラと数字が増える演出(duration: 1.5s)。
*   **Gauge Fill**:
    *   エネルギーゲージが、液体が注がれるように増える。

## 5. 技術選定
*   `react-native-reanimated`: 60fpsのスムーズなアニメーション実装。
*   `lottie-react-native`: 複雑なベクターアニメーション（ローディング、リザルト演出）の再生。
