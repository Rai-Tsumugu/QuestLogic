# 08_parent_dashboard.md

## 概要
親が子供の学習状況を把握し、AI分析結果を確認・承認するためのインターフェース仕様。
シナリオにおける「職場での安心感」「帰宅後のポジティブなコミュニケーション」を支援する。

## UI / 機能詳細

### 1. 通知 (Real-time Notification)
*   **Event**: 子供がクエスト完了（After撮影 & AI分析終了）時。
*   **Message**: 「健太くんが数学のクエストを完了しました！ AI評価: 85点（丁寧ボーナスあり）」
*   **Payload**: `{ questId, childId, score, aiSummary }`
*   **Direct Action**: 通知から直接「承認」や「スタンプ送信」ができる（OS機能活用）と望ましい。

### 2. クエスト詳細・承認画面 (Approval View)
*   **Before/After Slider**:
    *   スライダーバーを左右に動かすことで、Before画像とAfter画像をワイプ比較できるUI。
    *   「どれだけ書いたか」が一目瞭然になる。
*   **AI Report Card**:
    *   **Score**: 大きく表示 (e.g. "85 Good!")
    *   **Highlights**: AIが検出した「褒めポイント」を箇条書き。
        *   ✅ 計算ミスを修正しています
        *   ✅ 図解での整理が見られます
*   **Parent Actions**:
    *   **Approve (承認)**: 「OK」ボタン。子供に通知が飛び、エネルギーが確定する（基本）。
    *   **Stamp (スタンプ)**: 「花丸」「すごい！」「お疲れ様」などのスタンプを選択して承認。
        *   効果: 親からのスタンプは「Parent Boost」としてエネルギー+10分（設定可）。
    *   **Reject (却下)**: 理由（「字が雑すぎる」「解き直しをしていない」）を選択して差し戻し。

### 3. ダッシュボード (Home)
*   **Child Status**: 「現在: 遊び中（残り15分）」のようにリアルタイムステータスを表示。
*   **Today's Log**: 今日の学習時間合計、獲得エネルギー合計。

## 受け入れ条件 (Acceptance Criteria)
1.  通知を受け取り、タップすると詳細画面が開くこと。
2.  スライダーUIで画像の比較がスムーズに行えること。
3.  「スタンプ承認」を行うと、子供側の獲得エネルギーにボーナスが加算されること。
