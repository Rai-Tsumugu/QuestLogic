# Week 3 Day 5: Unlock Mock (IoT連携準備)（当日運用版）

## 作業時間目安
- 想定: 3-4時間
- 実行主体: 作業者

## 本日のゴール
1. ホーム画面で現在の「エネルギー残高」が表示される。
1. 「遊ぶ（Unlock）」ボタンを押すと、エネルギーを消費して「解錠中」状態になる。
1. タイマー（遊び時間）が作動する。

## 本日の重点差分
- 残高不足時Disabled状態と消費トランザクション整合を確認する。

- [ ] ログ取得手段確認: Metro, Firebase Console, Functionsログを閲覧できる

- [ ] 実施操作: Energy Hook。実施内容: Firestore `energy_transactions` を集計し、現在の残高時間を計算するHook (`useEnergy`) を実装。 / ホーム画面右上に電池アイコンと共に表示。
- [ ] コマンド/操作: firebase emulators:start（必要に応じて） / firebase deploy --only 対象サービス
- [ ] 期待結果: Energy Hook に関する主導線がエラーなく完走し、異常系は意図した挙動になる。
- [ ] 証跡 (EVID-16.1): スクリーンショット、ログ抜粋、DB/Storage確認パスを記録する

### STEP-16.2 Unlock UI
- [ ] 実施操作: Unlock UI。実施内容: 大きな「UNLOCK」ボタン。 / 押下時のアニメーション（鍵が開く音、アニメ）。
- [ ] コマンド/操作: 対象機能を実機で操作し、Metroログ/Consoleログを採取
- [ ] 期待結果: Unlock UI に関する主導線がエラーなく完走し、異常系は意図した挙動になる。
- [ ] 証跡 (EVID-16.2): スクリーンショット、ログ抜粋、DB/Storage確認パスを記録する

### STEP-16.3 Play Timer

- [ ] コマンド/操作: 対象機能を実機で操作し、Metroログ/Consoleログを採取
- [ ] 期待結果: Play Timer に関する主導線がエラーなく完走し、異常系は意図した挙動になる。
