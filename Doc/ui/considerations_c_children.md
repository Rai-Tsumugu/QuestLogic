# C. Children画面 (子供向けRPG UI) に関する詳細考慮項目

「勉強のツール」であることを忘れさせ、「これはゲームの冒険の準備である」と子供に認識させるための演出と操作性のチェックリストです。

## 1. 直感性とアクセシビリティ
- [ ] 漢字が読めない、または読むのが面倒な年齢（低学年）でも、アイコンや色だけで「次に何を押せばいいか」が分かるか？
- [ ] ボタン（とくに「クエスト開始」「写真をとる」等）のタップ領域は十分に大きいか？
- [ ] 重要でない設定項目（親との切り替えボタンなど）は、誤タップしない位置（画面上部など）に配置されているか？

## 2. ゲーミフィケーションの実感
- [ ] 自分のレベル、蓄積された魔法のエネルギー（遊び時間）は常にステータスとして可視化されているか？
- [ ] 報酬をもらう瞬間（AIからの一次報酬、親の承認後の二次報酬）に、宝箱が開く、ファンファーレが鳴る等の「気持ちいい演出」が組み込まれているか？
- [ ] 感想文の入力画面は、「冒険日誌を書く」のようなゲームのフレーバーに落とし込まれているか？

## 3. カメラUIという「難所」のサポート
- [ ] 水準器機能（スマホが水平になったら枠が光る等）は、子供にとって「ミニゲーム」のように楽しく合わせられる演出か？
- [ ] After撮影時の「ゴースト機能表示（Before画像を半透明で重ねる）」は、ズレが子供にも直感的にわかるUIになっているか？
- [ ] ブレて撮影してしまった際に「再撮影」を促すボタンは分かりやすいか？

## 4. AIフィードバックの心地よさ
- [ ] AI（相棒キャラクター、師匠など）からのメッセージは、画面の中でキャラクターが本当に喋っているようなUI（吹き出しなど）になっているか？
- [ ] ネガティブな結果（サボり判定など）を伝える際、冷たいエラーではなくキャラクターからの「少し残念なお知らせ・励まし」として表現できているか？
