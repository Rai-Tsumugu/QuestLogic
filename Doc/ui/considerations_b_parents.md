# B. Parents画面 (親向けダッシュボード) に関する詳細考慮項目

「疲れて帰ってきた親」や「仕事の合間に確認する親」にとって、直感的でノイズがなく、かつ安心感を与えられるUIにするためのチェックリストです。

## 1. 情報の優先順位とファーストビュー
- [ ] アプリを開いて最初の画面（ファーストビュー）で、「子供が今なにか対処を求めているか（承認待ちがあるか）」が一目でわかるか？
- [ ] 「今日の学習時間」や「現在の獲得エネルギー」は即座に確認できる位置にあるか？
- [ ] 兄弟姉妹など、子供が複数いる場合の子ユーザーの切り替えはスムーズか？

## 2. 効率的な承認フロー (Approval UX)
- [ ] クエストの詳細（Before/After画像、AI評価、子供の感想）を見るために必要なタップ数は最小限に抑えられているか？
- [ ] **Before/Afterのスライダー比較UI**は、親指が届く位置で操作しやすく、差分が直感的に理解できるか？
- [ ] 承認（スタンプを送る）や却下のアクションボタンは、明確に押し間違えない位置と色で配置されているか？
- [ ] 一括承認（もし複数溜まっていた場合）のようなショートカット機能は必要か？

## 3. 感情デザイン・コミュニケーション
- [ ] AIのレポートは、単なる減点法ではなく「ここを褒めてあげてください」というポジティブな行動を促す表現になっているか？
- [ ] スタンプやボーナス付与を通じて、子供とのコミュニケーションが楽しくなるような気持ちの良い演出（紙吹雪、完了チェックの小気味良いアニメーションなど）はあるか？

## 4. 設定と管理 (Settings)
- [ ] 報酬の「AI/親 配分比率」設定などの複雑な機能は、説明を読まなくてもスライダーやトグル等で直感的に変更できるか？
- [ ] PINコードの再発行やパスワード変更など、セキュリティに関わる導線は明確か？
