# デザイン提案書 (UI/UX Design Proposal) - 仮版

本ドキュメントは、「SmartStudy Gate」のUIデザインおよびUXフローの提案書です。
ユーザーシナリオに基づき、親と子で全く異なる世界観（デュアルインターフェース）を採用することで、それぞれのニーズを満たすデザインを目指します。

---

## 1. デザインコンセプト (Design Concept)

### 1.1 全体方針: Dual World (二つの世界)
ターゲットユーザーが全く異なるため、あえて統一感を排除し、アプリ内で完全に分離された2つの世界観を構築します。

| ユーザー層 | 世界観テーマ | キーワード |
| :--- | :--- | :--- |
| **子供 (Player)** | **Retro RPG Adventure** | 冒険、ドット絵、ワクワク、達成感、魔法 |
| **親 (Manager)** | **Clean Modern Dashboard** | 清潔感、信頼、可視化、効率、モニタリング |

### 1.2 子供向けデザイン: "The Quest"
*   **スタイル**: 8bit/16bit風のドット絵グラフィック。
*   **フォント**: ピクセルフォント（美咲フォント等）をメインに使用。
*   **演出**:
    *   画面遷移は暗転やワイプ演出を入れる。
    *   ローディングはキャラクターが走るアニメーション。

### 1.3 親向けデザイン: "The Gatekeeper"
*   **スタイル**: iOS/Material Designに準拠したフラットでミニマルなデザイン。
*   **フォント**: システムフォント (San Francisco / Roboto)。太字で階層を強調。
*   **演出**:
    *   情報はカード形式で整理。
    *   グラフや数値は大きく、直感的に。
    *   色は「安心感」を与える青・緑を基調に。

## 2. カラーパレット (Color Palette)

### Child UI (Retro Fantasy)
冒険のワクワク感を演出する高彩度な配色。

| Role | Color Code | Description |
| :--- | :--- | :--- |
| **Primary** | `#FFD700` (Gold) | 報酬、コイン、重要なボタン |
| **Secondary** | `#4169E1` (Royal Blue) | メニュー背景、マジックゲージ |
| **Success** | `#32CD32` (Lime Green) | クエストクリア、体力満タン |
| **Danger** | `#FF4500` (Orange Red) | 時間切れ、警告 |
| **Background** | `#2F4F4F` (Dark Slate) | ダンジョン風の暗めの背景 |

### Parent UI (Reliable Tech)
目に優しく、情報の視認性を最優先した配色。

| Role | Color Code | Description |
| :--- | :--- | :--- |
| **Primary** | `#007AFF` (System Blue) | アクションボタン、リンク |
| **Background** | `#F5F5F7` (Off White) | 全体の背景 (iOS like) |
| **Surface** | `#FFFFFF` (White) | カード背景、モーダル |
| **Before** | `#A9A9A9` (Gray) | Before画像枠、未完了 |
| **After** | `#34C759` (System Green) | After画像枠、完了・承認 |

## 3. 画面レイアウト案 (Screen Layouts)

### 3.1 Child: Home (Quest Board)
RPGの「酒場」や「ギルド」のような掲示板画面。

*   **Header**:
    *   Left: キャラクターアイコン & Lv.
    *   Right: **Energy Gauge** (電池アイコン + 残り時間)。
*   **Main**:
    *   **"Today's Quest" Board**: 巻物風のUIに「今日の宿題」が表示されている。
    *   クエストごとに「難易度（★の数）」や「報酬（宝箱アイコン）」を表示。
*   **Bottom Action**:
    *   大きな **[START]** ボタン（剣のアイコン）。

### 3.2 Child: Camera (Concentration Mode)
ゲームボーイの画面のような枠を付け、没入感を高める。

*   **Overlay**:
    *   四隅にレトロな装飾枠。
    *   中央に「ターゲット（グリッド）」表示。
*   **Feedback**:
    *   傾き検知時に「傾いてるぞ！」とドット絵キャラが吹き出しで警告。（AIで処理しやすい画角に誘導）
*   **Timer**:
    *   デジタル数字ではなく、砂時計やロウソクのアニメーションで時間を表現。

### 3.3 Child: Result (Victory!)
戦闘終了後のリザルト画面。

*   **Animation**:
    *   宝箱が開く → 数字が飛び出す。
*   **AI Comment**:
    *   王様や師匠キャラからの「お告げ」として表示。
    *   「よくやった！特にここ（図解）が素晴らしいぞ！」
    *   「ここをこうすれば報酬をはずむのじゃが…」
*   **Stamp**:
    *   親からの承認スタンプが届くと、空から降ってくる演出。

### 3.4 Parent: Dashboard
情報の網羅性を重視したリスト表示。

*   **Top Card**:
    *   子供のステータス（アイコン + 状態テキスト）。
    *   「現在: ゲーム中（あと15分）」などのリアルタイム表示。
*   **Timeline**:
    *   時系列でイベントを表示。
    *   [16:30] 宿題完了 (AI: 85点) [確認する]
    *   [16:35] ゲーム開始
*   **Fab (Floating Action Button)**:
    *   [+] 設定変更、手動ロック解除などの緊急アクション。
    *   [承認 (Stamp)] [却下 (Comment)] ボタンを固定表示。
