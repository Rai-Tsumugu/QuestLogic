# A. UI Image (インスパイア・アセット) に関する詳細考慮項目

親（モダン・マテリアル/iOS風）と子（16bit RPG風）という相反する世界観を1つのアプリに同居させるための設計チェックリストです。

## 1. デザインシステムの分離と管理
- [ ] UIコンポーネント（ボタン、入力項目、カードなど）は、親用と子用で完全にコンポーネントを分けるか？
- [ ] 共通で使うコンポーネント（もしあれば）は、テーマ切り替えで対応可能か？
- [ ] ピクセルアートの画像アセットは、各種解像度（Retina等）でぼやけずに表示されるようスケーリング設定（`image-rendering: pixelated`等）されているか？

## 2. カラースキームの確立
- [ ] **子供チームのカラー**: 「報酬・成功」「警告・時間切れ」「背景（ダンジョン等）」のメインカラーコードは定まっているか？
- [ ] **親チームのカラー**: 「安心感・信頼」「承認待ち・アラート」を示すカラーコードは定まっているか？
- [ ] 両方のUIモードにおいて、「テキストの可読性（コントラスト比）」は確保されているか？

## 3. タイポグラフィとフォント
- [ ] 子供画面用のピクセルフォント（例：美咲フォント等）は、十分な漢字をサポートしているか？
- [ ] ピクセルフォントによる文字化けや、長文（AIからのフィードバック等）が読みにくくなる問題への対策はあるか？
- [ ] カスタムフォントの読み込みによる初期表示遅延（FOUT/FOIT）の対策はできているか？

## 4. アイコンとイラスト（トーン＆マナー）
- [ ] 子供画面のアイコンは、世界観（剣、盾、ポーション、宝箱など）に沿ったメタファーになっているか？
- [ ] 親画面のアイコンは、一般的で普遍的なシステムアイコン（歯車、ベル、チェックマーク等）を使用しているか？
- [ ] ローディング時のスピナー（ぐるぐる）も、子供向けは走るスライムや砂時計などに変更されているか？
