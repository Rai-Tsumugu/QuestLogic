# E. 機能配置 (Feature Placement & Layout) に関する詳細考慮項目

スマホ端末の画面サイズや、ユーザーの手の大きさに合わせた、使いやすさ（アクセシビリティとエルゴノミクス）に関するチェックリストです。

## 1. アクションボタンの配置（親指可動域の考慮）
- [ ] 子供用カメラの「シャッターボタン」は、両手持ちでも片手持ちでも押しやすい位置（主に下部中央または右下）にあるか？
- [ ] 親用の「承認 (Approve)」「却下 (Reject)」ボタン群は、画面下部に固定（Sticky）されており、スクロール中でも常にアクセス可能か？
- [ ] フッターナビゲーション（タブバー等）の各アイコン領域は十分なタップエリア（最小44x44pt程度）を確保しているか？

## 2. 画面レイアウトの階層化
- [ ] **グローバルナビゲーション**（ホーム、設定、ログ等への移動）はどこに配置しているか？
    - 親は下部タブ、子供は隠しメニューまたはアイコンボタンなど、それぞれの振る舞いに合っているか？
- [ ] 「戻る」「閉じる」ボタンの配置はOS標準のルール（iOSなら左上やスワイプエッジ、Androidならシステムバック）と矛盾しないか？

## 3. モーダル / ボトムシートの使い分け
- [ ] 画面を完全に切り替える（Push遷移）べき内容と、ダイアログ・ボトムシートで済ませるべき内容の切り分け基準は定まっているか？
    - *例: 親の「報酬比率設定」変更はボトムシートでサッと変更できるのが望ましいか？*
    - *例: 子供の「感想入力」は専用のフルスクリーンダイアログで没入させるか？*
- [ ] キーボード展開時に、入力中のテキストフィールドや決定ボタンが隠れないようなスクロール/配置制御はされているか？

## 4. デバイス・レスポンシブ対応
- [ ] 最近の縦長スマホ（アスペクト比 19.5:9 など）の上下セーフエリア（ノッチ、ホームインジケーター）に要素が被らない配置となっているか？
- [ ] タブレット等の大画面デバイスで利用された場合、ただ間延びするのではなく、最大幅の制限（MAX-WIDTH）やグリッドのカラム数変更などの対策を入れるか？
